# Trained classifier and OOD evidence

## What is in production

The production ML layer has one bounded job: distinguish
`suction_restriction` from `discharge_restriction` on the SKAB pump after the
physics layer cannot separate them. It is not a universal fault classifier.

- Model: Extra Trees (`extra_trees-skab-restriction-v1`).
- Artifact: `backend/data/models/skab_restriction.joblib`.
- Training report: `backend/data/models/skab_restriction_report.json`.
- Reproducible trainer: `backend/data/train_fault_classifier.py`.
- Runtime: `backend/app/ml_classifier.py`.
- Source: SKAB v0.9 commit `b2c0d46c2971dcbfe71e26087b6d231998bb91c2`, GPL-3.0.

## Leakage-safe split

Training uses 17 independent valve experiments:

- suction: 15 `valve1` experiments, excluding frozen `valve1/2`;
- discharge: `valve2/2` and `valve2/3`.

It creates 510 labelled 30-reading windows. Multiple windows from one run are
allowed for training, but splitting is always by the complete experiment name.
No window from a held-out experiment appears in its training fold.

The frozen final restriction test is never used for model choice, probability
calibration, confidence threshold, or OOD threshold:

- `valve1/2.csv` — suction restriction;
- `valve2/0.csv` — discharge restriction;
- `valve2/1.csv` — discharge restriction.

## Features available at decision time

For each of six SKAB signals—vibration, current, pressure, motor temperature,
fluid temperature, and flow—the detector stores:

- mean, median and MAD;
- half-window drift and peak-to-peak range;
- first-quarter mean, last-quarter mean, delta and percentage delta;
- linear slope;
- volatility percentage and derivative standard deviation.

These are computed only from the current and preceding 30 readings. No future
row or evaluation label reaches production inference.

## Model selection and probability calibration

Extra Trees and random forest candidates are compared using
leave-one-physical-experiment-out predictions at the production detector's
trigger context. Extra Trees has 1.0 grouped balanced accuracy in the recorded
development run and is selected without looking at the frozen test.

An isotonic calibrator maps the model's raw discharge probability using only
out-of-fold window predictions. The confidence acceptance threshold is chosen
from grouped out-of-fold trigger predictions to target at least 95% selective
accuracy while maximizing episode coverage. The resulting threshold is 0.50.

This small, imbalanced calibration set—15 suction experiments and two discharge
experiments—is a serious limitation. A reported calibrated probability is a
development estimate, not a field reliability probability.

## Genuine OOD handling

The artifact contains an IsolationForest trained on in-distribution restriction
windows. For every training experiment, an OOD score is generated by a model
that did not see that entire experiment. The acceptance threshold is the 10th
percentile of those leave-one-experiment-out ID scores: `0.024273`.

Runtime abstains when either:

1. the exact six-signal feature contract is missing—schema OOD; or
2. the learned novelty score is below the calibrated ID threshold—distribution
   OOD; or
3. calibrated class confidence is below its acceptance threshold.

Same-schema OOD evaluation uses 14 different SKAB experiments: fluid
leak/addition, rotor-imbalance variants, water-level changes, cavitation, and hot
water. None trains the restriction classifier or tunes the threshold. Current
result: AUROC 1.0 and 14/14 rejected. This is a real learned novelty test, but
still from the same laboratory corpus.

CWRU has an entirely different bearing signal roster. The restriction model
marks it schema-OOD before inference; the general deterministic bearing
signature can still handle that supported class.

| `valve1/2` | suction | suction | 1.000 | 0.0500 |
| `valve2/0` | discharge | discharge | 0.553 | 0.0583 |
| `valve2/1` | discharge | discharge | 1.000 | 0.0632 |

Frozen restriction result: 3/3 overall, 100% coverage, 100% selective accuracy.
Always quote `n=3`; this is strong regression evidence, not a general guarantee.

## The rejected predecessor

The earlier `experiment_skab_classifier.py` attempted four-way classification
of rotor imbalance, cavitation, suction restriction and discharge restriction
from a single first-trigger summary. Its best model scored 3/5 and classified
`valve2/0` as suction restriction with about 0.95 confidence. It remains in the
repository as negative evidence and is not loaded by production.

The important engineering decision was not to tune that broad model against
the five failures. Instead, the production question was narrowed to the only
classification pair where ML adds value, and OOD rejection was made a first-
class requirement.

## What still needs external validation

- more independent discharge and cavitation experiments;
- natural customer fault-onset sequences instead of only lab testbeds;
- a locked external calibration/test set from another site;
- commercial data-rights clearance for every production training corpus;
- monitoring for feature drift, acceptance rate, selective accuracy, and human
  corrections after deployment.
